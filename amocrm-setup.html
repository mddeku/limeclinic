Интеграция AmoCRM с сайтом Lime Clinic

Это руководство поможет вам настроить автоматическую передачу заявок с сайта в AmoCRM CRM. Все заявки из формы записи, WhatsApp и Instagram будут автоматически попадать в воронку «Запись пациентов».

1

Регистрация в AmoCRM

Создайте аккаунт и выберите подходящий тариф

Шаги регистрации

  1. Перейдите на amocrm.ru и нажмите «Попробовать бесплатно» — 14 дней пробного периода
  2. Выберите тариф: рекомендуем «Профессиональный» — он включает автоматизацию, необходимую для медицины
  3. Укажите поддомен, например: limeclinic → ваш CRM будет на limeclinic.amocrm.ru
  4. Заполните данные организации: «Lime Clinic», сфера — «Медицина / Здравоохранение»
  5. Подтвердите email и войдите в систему
После регистрации сразу запишите ваш поддомен — он понадобится для всех API-запросов и файла .env
2

Создание воронки продаж (Pipeline)

Настройте этапы работы с пациентами

Создание воронки «Запись пациентов»

  1. Перейдите: Настройки → Воронки и статусы
  2. Нажмите «Добавить воронку» и назовите её «Запись пациентов»
  3. Добавьте следующие этапы в порядке слева направо:

Этапы воронки

📋 Новая заявка
📞 На связи
✅ Запись подтверждена
🏥 Принят
🎯 Успешно
❌ Отказ

Получить pipeline_id через API

После создания воронки получите её ID через API и добавьте в .env:

curl — получить ID воронок 
curl -X GET \
  https://ВАШ_ПОДДОМЕН.amocrm.ru/api/v4/leads/pipelines \
  -H "Authorization: Bearer ВАШ_ТОКЕН"

# В ответе найдите вашу воронку и скопируйте поле "id":
# {"id": 12345, "name": "Запись пациентов", ...}
# AMOCRM_PIPELINE_ID=12345

Название поля Тип Значения / Формат

Создание интеграции (OAuth2)

Зарегистрируйте приложение для API-доступа

Регистрация интеграции в AmoCRM

  1. Перейдите: Настройки → Интеграции → Создать интеграцию
  2. Название: «Сайт Lime Clinic»
  3. Описание: «Передача заявок с сайта limeclinic.kz в CRM»
  4. Права доступа: отметьте contacts (чтение + запись) и leads (чтение + запись)
  5. Redirect URI: укажите https://ваш-сайт.kz/api/amocrm/callback
  6. Нажмите «Сохранить» — получите Client ID и Client Secret
  7. Скопируйте оба ключа в файл .env

Пример .env с полученными данными

.env 
# AmoCRM OAuth2
AMOCRM_SUBDOMAIN=limeclinic
AMOCRM_CLIENT_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
AMOCRM_CLIENT_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
AMOCRM_REDIRECT_URI=https://limeclinic.kz/api/amocrm/callback
AMOCRM_CODE=           # заполнить на шаге 5
AMOCRM_PIPELINE_ID=12345  # ваш ID воронки
AMOCRM_STATUS_NEW=67890  # ID статуса "Новая заявка"

# Сервер
PORT=3000
ALLOWED_ORIGIN=https://limeclinic.kz

Получение токена авторизации

Однократная процедура OAuth2 авторизации

OAuth2 Flow для AmoCRM

1. Открыть URL
авторизации
2. Нажать
«Разрешить»
3. Получить code
в redirect URL
4. Сервер обменяет
code на токены

URL для авторизации

Откройте этот URL в браузере (замените ВАШ_CLIENT_ID на ваш реальный ID):

URL авторизации 
https://ПОДДОМЕН.amocrm.ru/oauth?
  client_id=ВАШ_CLIENT_ID&
  mode=post_message&
  redirect_uri=ВАШ_REDIRECT_URI

Автообмен code на токены

  1. После нажатия «Разрешить» браузер перенаправит вас на redirect URI с параметром ?code=XXXXXX
  2. Скопируйте значение code из адресной строки
  3. Вставьте его в .env: AMOCRM_CODE=XXXXXX
  4. Запустите сервер — он автоматически обменяет code на access_token и refresh_token
  5. Токены сохранятся в tokens.json и будут автоматически обновляться
Код авторизации (code) однократный и истекает через 20 минут. Используйте его сразу. После первого обмена файл tokens.json содержит долгосрочные токены — code больше не нужен.

Запуск сервера

Установите зависимости и запустите Node.js сервер

Команды запуска

bash — первый запуск 
# 1. Перейдите в папку проекта
cd /путь/к/lime-clinic/

# 2. Скопируйте пример .env
cp .env.example .env

# 3. Откройте .env и заполните все переменные
nano .env   # или любой редактор

# 4. Установите зависимости
npm install

# 5. Запустите сервер (production)
npm start

# Или в режиме разработки (автоперезапуск)
npm run dev

Тест endpoint через curl

curl — тест отправки заявки 
# Проверьте health check
curl http://localhost:3000/api/health

# Отправьте тестовую заявку
curl -X POST http://localhost:3000/api/amocrm/lead \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Тестовый Пациент",
    "phone": "+77077787890",
    "service": "Консультация уролога",
    "doctor": "Жанбырбекұлы Уланбек",
    "date": "2025-02-01",
    "time": "10:00",
    "source": "test"
  }'

# Ожидаемый ответ:
# {"success": true, "leadId": 12345}
Для продакшн-деплоя рекомендуем использовать PM2: pm2 start server.js --name lime-clinic. Это обеспечит автоматический перезапуск при сбоях и ведение логов.

При создании новой сделки SMS/WhatsApp пациенту: «Ваша заявка принята. Мы свяжемся с вами в течение 15 минут»

Часто задаваемые вопросы

Можно ли использовать бесплатный тариф AmoCRM?

Бесплатного тарифа в AmoCRM нет — только 14-дневный пробный период. Минимальный платный тариф «Базовый» подойдёт для начала, но для автоматизации (роботы, триггеры) потребуется тариф «Профессиональный». Начните с пробного периода, чтобы всё настроить и протестировать.

Как долго хранятся данные пациентов в AmoCRM?

AmoCRM хранит данные неограниченное время пока активна подписка. При отмене подписки предоставляется 60 дней для экспорта данных. Рекомендуется регулярно делать экспорт контактов и сделок в CSV как дополнительный бэкап.

Соответствует ли AmoCRM требованиям казахстанского законодательства о персональных данных?

AmoCRM хранит данные на серверах в России, что соответствует требованиям для пользователей из СНГ. По Закону РК «О персональных данных» вы обязаны получить согласие пациента на обработку данных (форма записи уже содержит такое согласие) и обеспечить безопасность хранения. Для клиник с высокими требованиями к локализации данных рассмотрите «Битрикс24» с хостингом в РК.

Можно ли интегрировать AmoCRM с медицинской МИС (системой управления клиникой)?

Да, AmoCRM имеет открытый REST API, через который можно синхронизировать данные с любой МИС (например, 1С:Медицина, MedElement, Medesk). При подтверждении записи в AmoCRM можно автоматически создавать визит в МИС через webhook или прямую интеграцию. Стоимость разработки такой интеграции — от 200 000 ₸ единовременно.
